1. The Basics

DNS

"a hierarchical and decentralized naming system for computers, services, or other resources connected to the Internet or a private network."

aka "the phone directory" of the Internet

Naming System

User need: I don't want to remember IP addresses, names are easier.

Hierarchical

Hierarchical

If I'm the owner of .be top-level domain (TLD)

  • I can manage/sell all the names in *.be
  • I can also delegate a subdomain like *.ac.be to universities

Top-level domains are supervised by the IANA (global organization)

Hierarchical

I bought a 2nd-level domain like odoo.com:

  • I can manage all the names in *.odoo.com
  • I can also delegate *.dev.odoo.com to the Odoo.sh team

and so on

Decentralized

Each operator maintain its own DNS zone:

  • a subset of the "address book"
  • the root zone of DNS is maintained by the IANA

Protocol operation

  • Called name resolution
  • Managed by your operating system
  • Over UDP, port 53

2. A tool you must know

dig

dig is a command-line tool to query the DNS

The command dig A odoo.com means:

  • ask the DNS resolver
  • What is the IP address of the server hosting odoo.com?
~$ dig A odoo.com

3. The "Records" family

Record?

  • a piece of information
  • with a type
  • specific to a domain

eg: The A record for odoo.com is 178.33.40.43

A & AAAA records

The IP address of the web server of the (sub)domain

  • A records for IPv4 addresses
  • AAAA records for IPv6 addresses

NS records

The name of the authoritative DNS servers for a domain

Redundancy is often set up, to avoid any failure

CNAME records

An alias pointing to another (sub)domain name

eg: www.myshop.com instead of myshop.odoo.com

Redundancy is often set up, to avoid any failure

PTR records

The hostname for a specific IP.

Used in "reverse DNS searches"

~$ dig -x 178.33.40.43
 
; <<>> DiG 9.14.9 <<>> -x 178.33.40.43
;; global options: +cmd

4. DNS in our support job

Basic configuration

  • Odoo maintains its own DNS servers
    • You can find them with dig NS odoo.com
  • SaaS and Odoo.sh platforms automate things
    • A records in *.odoo.com / *.dev.odoo.com
    • MX records for incoming emails
    • cleaning when the database is dropped

Custom domain names

  • There is a documentation
  • Must be a CNAME redirection pointing to mydatabase.odoo.com
    • Customers are owners of their DNS zones
    • This is why we cannot configure them
    • Most of domain name resellers document how to do this

Custom domain names

  • Our servers host many databases on the same machine/IP
    • DNS redirection is not enough
    • We must tell which database should be used
    • Config in SaaS user portal or Odoo.sh branch settings

Custom domain names

Custom domain names

Multi-website

  • Repeat previous steps for each custom domain
  • In the website settings, configure a domain for each website

Common traps

  • Single page with iframe tag...
  • HTTP 301 redirection instead of CNAME record
  • CDNs like Cloudflare

These configuration schemes can be suggested by DNS providers

Mail specific configs

  • Generally depend on the domain name used in the From email address
  • SPF & DKIM configuration

SPF

  • A TXT record with the SPF policy
  • The IPs of the servers allowed to send emails from this domain
  • Customers should include our SPF record in theirs

Cache issues

Each DNS response comes with a TTL value, used in various cache mechanisms

*.odoo.com always uses 600 seconds

Issue location?

  • browser DNS cache
  • restart your Internet "box"
  • corporate DNS resolvers
  • ISP DNS resolvers
  • any network device that would be misconfigured...

Hosts files

  • Signs from the past...
    • Were the only resource available before DNS (1982-1983)
  • Can be used to hack the DNS & debugging multi-websites locally
    • /etc/hosts
    • C:\Windows\System32\drivers\etc\hosts
